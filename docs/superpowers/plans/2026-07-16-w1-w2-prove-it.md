# W1 + W2 "Prove It" Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Resolve PR #25 (013 config coherence) with a recorded decision, and fatten the benchmark from 2 to 14 cases covering all five promised failure patterns, with regression thresholds on TG-G3/TG-G4/TG-G5 and a user-visible scorecard in the README.

**Architecture:** The eval harness (`packages/eval`) already runs the real `scan → gates` pipeline over `benchmark/cases/<name>/{expected.json, repo/}` fixtures and scores by the `(gate_id, path, tier)` match key. This plan adds cases (data), one small gate change (G4 consumes the project map's `data_access` evidence at `medium` confidence so it can only ever be `suspected` until W3's windowed detection), extended thresholds, and README scorecard rendering. No new architecture.

**Tech Stack:** TypeScript, Node 22, pnpm workspaces, Vitest, existing `@aker-build/eval` + `@aker-build/gates` packages.

## Global Constraints

- Constitution hard rules: read-only detection; no secrets stored or printed (fixture "secrets" are variable names only, never values); synthetic fixtures only — no Retail Tower / ERPNext logic.
- Confidence honesty: `confirmed` = at least one `high`-confidence evidence (`packages/gates/src/confidence.ts` max rule); a heuristic that cannot prove absence must emit `medium` (→ `suspected`).
- Git: stage NAMED files only (never `git add -A` / `git add .`). Commit format `<type>: <description>`. No push / PR / merge without the user's explicit request.
- Work on a new branch `w2-prove-it-benchmark` cut from up-to-date `main`.
- Verification before any "done" claim: `pnpm test` and `pnpm typecheck` at repo root must pass.
- Benchmark ground truth rule: `expected.json` encodes GROUND TRUTH (what a correct engine should find), never current-engine behavior. A known engine gap shows up as an honest FP/FN in the scorecard, not as a doctored expectation.

**Interfaces:**
- Consumes: `ctx.projectMap.data_access?: Evidence[]` (populated by `packages/scanner/src/detect/data-access.ts`; each item: `{ type: "line", path, line, signal: "tenant_scoped" | "no_tenant_filter", confidence: "high" }`). Helper `lineEvidence(path, line, signal, confidence)` and `risk(id, severity, evidence)` from `packages/gates/src/gates/helpers.ts`.
- Produces: G4 emits one `risk` finding per `no_tenant_filter` site: severity `"high"`, evidence confidence `"medium"` (→ tier `suspected` via `confidenceTier`). Benchmark tasks 3 and 6 rely on this tier being `suspected`.

Why `medium`, not `high`: the detector's same-line heuristic (D3 in the spec) cannot prove the filter is absent — it may sit on the next line (multi-line Prisma). W3 upgrades this to `high` once windowed detection can actually prove it. Encoding uncertainty in the tier is the P2 honesty rule.

### Task 6: Extend regression thresholds to G3/G4/G5

**Files:**
- Modify: `benchmark/thresholds.json`

**Interfaces:**
- Consumes: `Thresholds` shape (`packages/eval/src/thresholds.ts`): `gate -> tier -> { min_precision?, min_recall? }`. A `null` metric (no data at that tier) never breaches — so `confirmed` floors on G5 are inert until a confirmed-tier G5 rule exists, and that is fine (they arm automatically when it does).
- Produces: the CI regression contract Task 7's README documents.

- [ ] **Step 1: Write the new thresholds**

Replace `benchmark/thresholds.json` with:

```json
{
  "TG-G3": {
    "confirmed": { "min_precision": 0.9, "min_recall": 0.85 },
    "suspected": { "min_recall": 0.85 }
  },
  "TG-G4": {
    "confirmed": { "min_precision": 0.9, "min_recall": 0.85 }
  },
  "TG-G5": {
    "confirmed": { "min_precision": 0.9, "min_recall": 0.85 },
    "suspected": { "min_recall": 0.85 }
  }
}
```

Deliberate shape: `suspected` gets recall floors only, never precision floors — the suspected tier is the honest-uncertainty channel (the multiline-tenant-scope FP lives there by design until W3), so flooring its precision would punish honesty. TG-G2 is absent: G2 emits only `needs_verification` in v0 (needs 007 diff evidence) and the harness scores only `risk` findings — recorded as a spec deviation in Task 7.
